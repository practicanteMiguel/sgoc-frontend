1
Estado de botones nunca se actualizaba visualmente
src/components/modulos/reports/activities/crew-modal.tsx
Severidad: Media Tipo: UX / Regresion silenciosa

El estado pendiente de las mutaciones se almacenaba en useRef<Set<string>>, que no es reactivo. Al llamar a pendingRef.current.has(emp.id) durante el render, React no re-renderizaba el componente al mutar el ref, por lo que los botones de toggle de empleados nunca mostraban el estado de carga. El usuario podia hacer clic varias veces sin feedback visual. 

// Antes (bug - ref no reactivo)
const pendingRef = useRef(new Set<string>())
function toggleEmployee(emp) {
  if (pendingRef.current.has(emp.id)) return
  pendingRef.current.add(emp.id)
  mutate(..., { onSettled: () => pendingRef.current.delete(emp.id) })
}
const isPending = pendingRef.current.has(emp.id) // siempre false en UI

// Despues (corregido - useState reactivo)
const [pendingIds, setPendingIds] = useState(new Set<string>())
function toggleEmployee(emp) {
  if (pendingIds.has(emp.id)) return
  setPendingIds(prev => new Set(prev).add(emp.id))
  const finish = () => setPendingIds(prev => {
    const next = new Set(prev); next.delete(emp.id); return next
  })
  mutate(..., { onSettled: finish })
}
2
Mutacion de variable fuera de useMemo (React Compiler)
src/components/modulos/reports/activities/charts/report-weekly-chart.tsx
Severidad: Baja Tipo: Incompatibilidad React Compiler

La variable acumuladora cumProj se declaraba fuera del useMemo y se mutaba dentro con +=. El React Compiler (activado por defecto en Next.js 16) detecta esta violacion de inmutabilidad y puede invalidar la memoizacion de forma impredecible. 

// Antes (violacion de inmutabilidad)
let cumProj = 0
const data = useMemo(() => rawData.map(d => {
  cumProj += d.dailyProj  // muta variable externa!
  return { ...d, projPct: cumProj }
}), [rawData])

// Despues (inmutable con reduce)
const data = useMemo(() => rawData.reduce<...[]>((acc, d) => {
  const prev    = acc[acc.length - 1]
  const cumProj = (prev?.projPct ?? 0) + d.dailyProj
  return [...acc, { ...d, projPct: Math.round(cumProj), ... }]
}, []), [rawData])

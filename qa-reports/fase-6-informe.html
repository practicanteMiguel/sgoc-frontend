generateWeekExcel - falta verificacion res.ok antes de arrayBuffer()
src/components/modulos/reports/activities/supervisor-activities-view.tsx : 150-153
Error silencioso Inconsistencia de patron Export Excel

La funcion generateWeekExcel descarga imagenes de actividades (antes/durante/despues) para embeber en el Excel. El codigo llama res.arrayBuffer() sin verificar primero res.ok. Si una URL de imagen devuelve 403 o 404 (por ejemplo, una URL firmada expirada), fetch resuelve correctamente pero retorna el body de error (HTML de la pagina de error). ExcelJS intenta insertar ese HTML como imagen PNG/JPEG y lanza una excepcion interna. El catch la absorbe silenciosamente. El patron correcto ya existe en fetchLogoBuffer (usado en dotacion-export.ts y generate-via-word.ts) que verifica !res.ok antes de llamar arrayBuffer(). 

// ANTES - supervisor-activities-view.tsx:148
          try {
            const res = await fetch(url)
            const buf = await res.arrayBuffer()   // error HTML si 404/403
            const ct  = res.headers.get('content-type') ?? ''

// DESPUES - patron consistente con fetchLogoBuffer
          try {
            const res = await fetch(url)
            if (!res.ok) continue               // skip imagen no disponible
            const buf = await res.arrayBuffer()
            const ct  = res.headers.get('content-type') ?? ''

handleGenerate y handleSave - catch vacio sin feedback al usuario
src/components/modulos/reports/activities/report-generate.tsx : 418-442
UX silencioso Sin toast de error Informes tecnicos

Ambas funciones tienen bloques catch {} completamente vacios. Si la API falla al crear o actualizar un informe tecnico, el spinner desaparece y el boton vuelve a aparecer sin ningun mensaje de error. El usuario no sabe si debe reintentar, si el problema es transitorio, o si sus datos se perdieron. Contraste con handlePdf en el mismo archivo que usa try/finally apropiadamente. 

// ANTES - handleGenerate (linea 426) y handleSave (linea 441)
    } catch {}  // usuario no sabe que fallo la operacion
    setGenerating(false)

// DESPUES - con toast de error en ambas funciones
    } catch {
      toast.error('Error al generar el informe')
    }
    setGenerating(false)

    // handleSave:
    } catch {
      toast.error('Error al guardar los cambios')
    }
    setSaving(false)
3
VoiceReportModal - sin handler onError en la mutacion de generacion IA
src/components/modulos/monitoring/voice/voice-report-modal.tsx : 103-114
Sin feedback de error Import toast faltante Modulo Monitoreo

generate.mutate llama a la API de IA para procesar transcripciones de voz y generar un informe estructurado. El segundo argumento de mutate solo tiene onSuccess. Si la API de IA falla (error de red, timeout, tokens insuficientes), generate.isPending vuelve a false, el spinner desaparece y el boton reaparece sin ningun toast ni mensaje. Adicionalmente, toast de sonner no estaba importado en este archivo, por lo que se agrego el import junto con el handler. 

// ANTES - voice-report-modal.tsx
// sin import de toast

    generate.mutate({ ids, title: resolvedTitle }, {
      onSuccess: async (data) => { ... },
      // falta onError
    })

// DESPUES
import { toast } from 'sonner'

    generate.mutate({ ids, title: resolvedTitle }, {
      onSuccess: async (data) => { ... },
      onError: () => toast.error('Error al generar el informe'),
    })

# Lo que parecia un problema Archivo Por que es correcto
1 Cast complejo de ExcelJS: (excelModule as { default?: ... }).default ?? excelModule dotacion-export.ts ExcelJS tiene exports tanto default como nombrados segun el bundler. El cast es necesario para compatibilidad con Webpack y Turbopack.
2 eslint-disable react-hooks/exhaustive-deps en filteredReports supervisor-vias-view.tsx:144 Patron deliberado documentado en Fase 5. La variable se asigna fuera del useMemo intencionalmente para control manual de dependencias.
3 Fetch secuencial de imagenes en bucle for await generate-via-word.ts:154 Las imagenes se deben insertar en el documento Word en orden. El procesamiento paralelo romperia el orden de los parrafos en el array body.
4 BASE = Date.UTC(2025, 11, 29) hardcodeado en multiples archivos report-generate.tsx:46, supervisor-activities-view.tsx:28 Fecha base del sistema de semanas fiscales del proyecto. Es una constante del dominio de negocio, no un magic number accidental. Usada consistentemente en todos los calculos de Gantt y barra de cumplimiento.
5 fetchBuf duplica la logica de fetchLogoBuffer requisicion-detail.tsx:184 Funcion auxiliar anidada dentro de exportExcel. La duplicacion es intencional para evitar un import adicional en una funcion standalone que no necesita el modulo completo de report-header.
